# Auto-Flashcards — Product Requirements Document (PRD)

> Документ описывает цели, пользователей, проблемы и ключевые требования проекта **Auto-Flashcards** — системы автоматической генерации флэш-карточек с помощью AI.  
> Цель — определить ценность продукта, его аудиторию и критерии успеха.

---

## 1. Общее описание проекта

**Название:** Auto-Flashcards  
**Категория:** Образование / EdTech  
**Тип продукта:** Веб-приложение на основе AI (NLP + RAG)  
**Цель:** Автоматизировать процесс создания и изучения учебных материалов через генерацию карточек “вопрос–ответ”.

---

## 2. Проблема и мотивация

Современные студенты сталкиваются с рядом проблем:

- ⏱️ **Проблема времени:** много учебных материалов, но не хватает времени на эффективное повторение.
- 🧠 **Проблема запоминания:** традиционные методы (чтение, конспекты) не обеспечивают долговременное усвоение.
- 💤 **Проблема мотивации:** монотонное заучивание не стимулирует интерес к предмету.
- 📄 **Проблема структурирования:** сложно выделить ключевые понятия из больших текстов.

Auto-Flashcards решает эти проблемы, создавая интеллектуальные карточки с интервальным повторением и автоматическим извлечением понятий.

## 3. Целевые сегменты пользователей

| Сегмент                            | Описание                                               | Цель использования                             |
| ---------------------------------- | ------------------------------------------------------ | ---------------------------------------------- |
| 🎓 **Студенты университетов**      | 18–25 лет, активно используют онлайн-обучение          | Повторение лекций, подготовка к экзаменам      |
| 👨‍💻 **Самообучающиеся специалисты** | Повышают квалификацию, изучают новые темы              | Быстрое усвоение информации с помощью карточек |

## 4. Основные Pain Points (Боли пользователей)

| №   | Pain Point                                      | Последствие                       |
| --- | ----------------------------------------------- | --------------------------------- |
| 1   | Трудно выделить главное из длинных PDF и лекций | Потеря времени и фокусировки      |
| 2   | Нет удобного инструмента для повторения         | Информация быстро забывается      |
| 3   | Создание карточек вручную занимает часы         | Пользователи забрасывают обучение |
| 4   | Сложно отслеживать прогресс и результаты        | Низкая мотивация и эффективность  |

| №    | Сценарий                             | Пользовательская ценность                      |
| ---- | ------------------------------------ | ---------------------------------------------- |

## 6. Value Proposition (Ценностное предложение)

> **Auto-Flashcards** — это AI-сервис, который превращает любые учебные материалы в персональные флэш-карточки с интервальным повторением.  
> Он помогает студентам и преподавателям **экономить время** и **улучшать запоминание** без ручного труда.

## 7. North Star Metric (Главная метрика успеха)

| Метрика                            | Цель                        | Формула / Пример                                       |
| ---------------------------------- | --------------------------- | ------------------------------------------------------ |
| **Retention Rate (7 дней)**        | Удержание пользователей     | % пользователей, вернувшихся к повторению через неделю |
| **Avg. Cards Learned per Session** | Эффективность обучения      | ≥ 30 карточек за сессию                                |
| **AI Accuracy**                    | Качество генерации карточек | ≥ 85% карточек признаны “корректными” пользователями   |

## 8. Функциональные требования (FR)

| №    | Требование                           | Описание                               |
| ---- | ------------------------------------ | -------------------------------------- |
| FR-1 | Автоматическая генерация карточек    | Генерация Q/A по тексту, PDF или видео |
| FR-2 | Поддержка AI-моделей NLP и OCR       | Извлечение текста и ключевых понятий   |
| FR-3 | Интервальное повторение (SRS)        | Алгоритм SM-2 / Anki-style             |
| FR-4 | Редактирование и управление наборами | CRUD-операции над карточками           |
| FR-5 | Ведение прогресса                    | Сохранение результатов повторений      |
| FR-6 | Экспорт и импорт наборов             | PDF / CSV / Share link                 |
| FR-7 | Мультиязычная поддержка              | Русский / Английский / Арабский        |

## 9. Нефункциональные требования (NFR)

| №     | Категория          | Требование                           |
| ----- | ------------------ | ------------------------------------ |
| NFR-1 | Производительность | Генерация ≤ 60 сек для файла ≤ 25MB  |
| NFR-2 | Надёжность         | Автосохранение при сбое              |
| NFR-3 | Удобство (UX)      | Минимум 3 клика до создания карточек |
| NFR-4 | Масштабируемость   | Обработка ≥ 1000 карточек в день     |
| NFR-5 | Совместимость      | Работа в браузере и мобильном виде   |
