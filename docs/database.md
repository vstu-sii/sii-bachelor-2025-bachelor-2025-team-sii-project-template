# Документация по схеме базы данных
## Система анализа отзывов

## Обзор

Эта база данных поддерживает комплексную систему анализа отзывов, которая интегрируется с Telegram для аутентификации и с множеством маркетплейсов для анализа отзывов о товарах. Система управляет пользователями, отслеживанием товаров, анализом тональности на основе ИИ и сравнительной отчетностью.

## Детальные спецификации таблиц

### 1. Таблица Пользователи

**Назначение**: Хранение учетных записей пользователей, аутентифицированных через Telegram Widget

| Поле | Тип | Описание | Отношение к сценариям использования |
|-------|------|-------------|-------------------|
| `telegram_id` | BIGINT | Уникальный идентификатор пользователя Telegram | Первичный ключ аутентификации |
| `telegram_username` | VARCHAR(255) | Имя пользователя Telegram с @ | Идентификация пользователя |
| `telegram_first_name` | VARCHAR(255) | Имя пользователя | Отображение в профиле |
| `registration_date` | TIMESTAMP | Время создания аккаунта | Аналитика пользователей |
| `last_login` | TIMESTAMP | Последний успешный вход | Управление сессиями |

**Индексы**:
- `idx_telegram_id` - Быстрый поиск пользователей при аутентификации
- `idx_registration_date` - Аналитика роста пользователей
- `idx_last_login` - Анализ активных пользователей

**Связанные сценарии использования**: 
- Сценарий 1: Регистрация через Telegram
- Сценарий 2: Вход через Telegram  
- Сценарий 3: Выход из системы

### 2. Таблица Проекты

**Назначение**: Управление товарами пользователей для анализа и отслеживания

| Поле | Тип | Описание | Отношение к сценариям использования |
|-------|------|-------------|-------------------|
| `marketplace_url` | VARCHAR(1000) | URL страницы товара | Идентификация товара |
| `product_name` | VARCHAR(500) | Название товара | Отображение в интерфейсе |
| `marketplace_type` | ENUM | Поддерживаемый маркетплейс | Выбор парсера |
| `current_rating` | DECIMAL(3,2) | Последний рейтинг | Быстрый обзор |
| `is_tracking_active` | BOOLEAN | Статус мониторинга | Управление отслеживанием |

**Ограничения**:
- `unique_user_product` - Предотвращение дублирования товаров у пользователя
- Внешний ключ на Пользователей с каскадным удалением

**Индексы**:
- `idx_user_id` - Быстрая загрузка товаров пользователя
- `idx_marketplace_url(255)` - Префиксный индекс для поиска по URL
- `idx_last_analysis_date` - Сортировка по последней активности

**Связанные сценарии использования**:
- Сценарий 4: Первый анализ (добавление нового продукта)
- Сценарий 5: Просмотр ранее проанализированных продуктов
- Сценарий 6: Просмотр конкретного продукта

### 3. Таблица Анализы

**Назначение**: Отслеживание отдельных сессий анализа с временными данными

| Поле | Тип | Описание | Отношение к сценариям использования |
|-------|------|-------------|-------------------|
| `analysis_type` | ENUM | Начальный, периодический или пользовательский | Контекст анализа |
| `period_start` | DATE | Начало периода анализа | Фильтрация по времени |
| `period_end` | DATE | Конец периода анализа | Основа для сравнения |
| `total_reviews_processed` | INT | Проанализированные отзывы | Отслеживание прогресса |
| `average_rating` | DECIMAL(3,2) | Рассчитанный рейтинг | Ключевой показатель |
| `rating_change` | DECIMAL(3,2) | Изменение относительно предыдущего | Анализ тенденций |
| `status` | ENUM | Состояние обработки | Обновления статуса в интерфейсе |

**Ограничения**:
- `CHECK (period_start <= period_end)` - Валидный диапазон дат
- Внешний ключ на Проекты с каскадным удалением

**Индексы**:
- `idx_period_range` - Эффективные запросы по диапазону дат
- `idx_analysis_date` - Сортировка по времени анализа
- `idx_rating_change` - Поиск значительных изменений

**Связанные сценарии использования**:
- Сценарий 7: Запустить новый анализ по датам
- Сценарий 8: Сравнить выбранные анализы

### 4. Таблица РезультатыАнализа

**Назначение**: Хранение детальных результатов анализа по аспектам

| Поле | Тип | Описание | Отношение к сценариям использования |
|-------|------|-------------|-------------------|
| `aspect_category` | ENUM | Предопределенные категории аспектов | Организованный анализ |
| `aspect_name` | VARCHAR(100) | Конкретное название аспекта | Детальная разбивка |
| `mentions_count` | INT | Общее количество упоминаний аспекта | Метрика важности |
| `sentiment_score` | DECIMAL(4,3) | Тональность от -1 до +1 | Количественная мера |
| `common_themes` | JSON | Частые темы | Качественные инсайты |
| `improvement_suggestions` | JSON | Предложения пользователей | Полезная обратная связь |

**Ограничения**:
- `unique_analysis_aspect` - Одна запись на аспект в анализе
- Внешний ключ на Анализы с каскадным удалением

**Индексы**:
- `idx_sentiment_score` - Поиск позитивных/негативных аспектов
- `idx_mentions_count` - Идентификация часто обсуждаемых аспектов

**Связанные сценарии использования**:
- Сценарий 6: Просмотр конкретного продукта (детальная аналитика)
- Сценарий 8: Сравнить выбранные анализы (аспектное сравнение)

### 5. Таблица ВзаимодействияLLM

**Назначение**: Журнал аудита для всех взаимодействий с моделью ИИ

| Поле | Тип | Описание | Отношение к сценариям использования |
|-------|------|-------------|-------------------|
| `interaction_type` | ENUM | Тип задачи ИИ | Категоризация функций |
| `prompt_text` | TEXT | Полный отправленный промпт | Отладка и улучшение |
| `llm_response` | JSON | Структурированный ответ | Хранение результатов |
| `model_used` | VARCHAR(100) | Идентификатор модели ИИ | Отслеживание стоимости и производительности |
| `total_tokens` | INT | Использование токенов | Расчет стоимости |
| `processing_time_ms` | INT | Метрика производительности | Данные для оптимизации |

**Индексы**:
- `idx_interaction_type` - Фильтрация по типу задачи
- `idx_created_at` - Анализ по времени
- `idx_success` - Мониторинг уровня ошибок

**Связанные сценарии использования**:
- Все сценарии анализа, связанные с обработкой ИИ

### 6. Таблица ОтчетыСравнения

**Назначение**: Хранение сгенерированных отчетов сравнения для пользователей

| Поле | Тип | Описание | Отношение к сценариям использования |
|-------|------|-------------|-------------------|
| `analysis_ids` | JSON | Массив сравниваемых анализов | Состав отчета |
| `comparison_data` | JSON | Структурированные результаты сравнения | Содержание отчета |
| `key_insights` | TEXT | Сгенерированная сводка | Быстрые инсайты |
| `visualization_data` | JSON | Данные для графиков и диаграмм | Отрисовка в интерфейсе |
| `share_token` | VARCHAR(100) | Токен публичного доступа | Общий доступ к отчетам |

**Индексы**:
- `idx_share_token` - Быстрый доступ к общим отчетам
- `idx_generated_at` - Свежесть отчетов

**Связанные сценарии использования**:
- Сценарий 8: Сравнить выбранные анализы

## Оптимизация производительности

### Критические индексы

```sql
-- Производительность аутентификации
INDEX idx_telegram_id (telegram_id) ON Пользователи
INDEX idx_session_token (session_token) ON СессииПользователей

-- Производительность пользовательского интерфейса  
INDEX idx_user_id (user_id) ON Проекты
INDEX idx_project_id (project_id) ON Анализы
INDEX idx_analysis_date (analysis_date) ON Анализы

-- Аналитические запросы
INDEX idx_period_range (period_start, period_end) ON Анализы
INDEX idx_sentiment_score (sentiment_score) ON РезультатыАнализа
INDEX idx_rating_change (rating_change) ON Анализы
```
