# 1. Введение
AutoPitch Deck Generator — интеллектуальная система, позволяющая автоматически создавать и улучшать презентации стартапов на основе исходных материалов пользователя.
Пользователь загружает существующую презентацию (например, в PowerPoint или PDF), а система с помощью искусственного интеллекта сокращает её до ключевых смыслов, улучшает структуру и автоматически применяет визуальный стиль в соответствии с профессиональными стандартами презентаций для инвесторов.
Цель проекта:
Автоматизировать процесс подготовки питч-деков для основателей стартапов, сократив время на дизайн и редактирование и повысив убедительность визуальной подачи.


# 2. Use‑cases

# UC1 — Загрузка презентации для улучшения

| Поле | Описание |
|------|-----------|
| Название | Улучшение существующей презентации |
| Акторы | Пользователь (основатель стартапа)| Система (AutoPitch Deck Generator) | AI-модуль |
| Предусловия | Пользователь аутентифицирован в системе | Имеет презентацию в формате PowerPoint (.pptx) или PDF | Система доступна и подключена к AI-модулю
| Основной сценарий |  1. Пользователь входит в систему и выбирает функцию «Улучшить презентацию».<br>2. Система предлагает загрузить файл (.pptx или .pdf).<br>3. Пользователь загружает документ.<br>4. Backend получает файл и отправляет его в модуль анализа структуры и содержания (AI Parser).<br>5. AI-модуль извлекает текст, изображения и структуру слайдов.<br>6. LLM-модуль анализирует содержание, сокращает избыточные тексты и формирует ключевые тезисы.<br>7. Система применяет выбранный визуальный стиль (по шаблону или сгенерированный автоматически).<br>8. Backend формирует обновлённую презентацию и возвращает пользователю файл для предпросмотра.<br>9. Пользователь может внести правки (текст/цвета/расположение).<br>10. Система экспортирует финальную версию. |
| Постусловия | Создана новая улучшенная презентация | Доступна функция предпросмотра и скачивания
# UC2 — Создание новой презентации с нуля

| Поле | Описание |
|------|----------|
| Название | Автоматическая генерация новой презентации |
| Акторы | Пользователь\| Система (AutoPitch Deck Generator)  \| LLM-модуль |
| Предусловия | Пользователь аутентифицирован  \| Доступен LLM-API для генерации текста |
| Основной сценарий | 1. Пользователь выбирает опцию «Создать с нуля».<br>2. Вводит краткое описание стартапа (название, цель, продукт, аудитория, бизнес-модель).<br>3. Backend отправляет данные в LLM-модуль для генерации структуры презентации (10–12 слайдов).<br>4. LLM-модуль возвращает структурированный текст по разделам (Проблема, Решение, Продукт, Рынок, Команда, Финансы и т.д.).<br>5. Система применяет шаблон дизайна.<br>6. Frontend визуализирует результат в интерактивном редакторе.<br>7. Пользователь может редактировать текст и менять стиль оформления.<br>8. Система экспортирует финальную версию в PowerPoint или PDF. |
| Постусловия | Создана готовая презентация | Сохранена копия в облаке пользователя |

# 3. Функциональные требования

**Аутентификация и управление пользователями**
- Регистрация и вход 
- Личный кабинет с историей проектов и презентаций

**Основные функции**
-	Загрузка существующей презентации (PDF, PPTX)
-	Автоматическое извлечение структуры и текста (AI Parser)
-	Оптимизация контента через LLM (сокращение текста, структурирование)
-	Применение визуального дизайна (AI Design Engine)
-	Возможность редактирования и предпросмотра результата
-	Экспорт презентации (PPTX, PDF)
-	Сохранение проектов в облаке


**AI-функциональность**
-	LLM-API для обработки и сокращения текста
-	AI-модуль для генерации новых слайдов
-	Генерация дизайна и цветовых схем на основе контекста презентации


**Хранение данных:**
-	Пользователи и их проекты
-	Загрузки и версии презентаций
-	Настройки шаблонов и фирменных стилей


**Интеграции:**
-	LLM API (OpenAI)
-	Облачное хранилище (Google Drive)
-	Внешние шаблоны дизайна (Figma API / шаблоны JSON)



# 4. Нефункциональные требования и метрики производительности

**Производительность:**
-	Время генерации новой презентации ≤ 45 секунд
-	Время улучшения существующей презентации ≤ 30 секунд
-	Время рендеринга предпросмотра ≤ 3 секунды
-	Время отклика Backend API ≤ 2 секунды


**Пропускная способность:** поддержка не менее 50 одновременных пользователей, запускающих анализ

**Доступность:** 
-	Uptime Backend ≥ 99%
-	Веб-интерфейс доступен ≥ 99.5%


**Надёжность:** 
- При сбое внешнего сервиса (NLP, LLM) система возвращает сообщение об ошибке ≤ 3 секунд
- Автоматическое восстановление после временной недоступности парсеров

**Использование ресурсов:** 
- Загрузка CPU ≤ 70% при пиковой нагрузке
- Использование памяти ≤ 75% при пиковой нагрузке

**Безопасность:**
•	HTTPS-шифрование
•	Валидация входных данных и загружаемых файлов
•	Ограничение размера файлов (до 100 МБ)
•	Безопасное хранение данных в облаке 

# 5. Технологический стек

**Frontend:** React/Node.js

**Backend:** Python/FastAPI + Celery + Redis

**Database:** PostgreSQL + Redis (кеширование)

**Интеграции:**
- Telegram Widget для аутентификации
- LLM API (анализ тональности,классификация аспектов,генерация рекомендаций по улучшению)

# 6. Архитектурные принципы

Используется модель C4 для описания архитектуры:  
- **Context Diagram**  
- **Container Diagram**  
- **Component Diagram** 

-	Separation of Concerns: чёткое разделение между frontend, backend, AI-сервисом и хранилищем данных
-	Fail-safe defaults: при ошибке AI-модуля система возвращает старую версию презентации с предупреждением
-	Event-driven архитектура: обработка долгих задач через очередь (Celery)
-	Modularity: возможность добавления новых шаблонов дизайна и AI-моделей без изменения ядра
-	Scalability: поддержка горизонтального масштабирования backend и AI-сервисов


# 7. Ограничения и допущения

-	Используются сторонние LLM-сервисы (не разрабатываются внутри проекта)
-	Поддерживаются только форматы .pptx и .pdf
-	Приложение работает в браузере (десктоп), адаптивность под мобильные устройства — в будущем релизе
-	Качество AI-оптимизации текста зависит от модели
-	Генерация дизайна носит рекомендательный характер, а не гарантирует уникальность
