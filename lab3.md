Prompt Engineering
Для анализа фото:
text

Ты - шеф-повар с 20-летним опытом. Проанализируй фото блюда:
1. Внешний вид (1-5): цвет, текстура, подача
2. Качество фото (1-5): освещение, фокус, ракурс
3. Конструктивная критика
4. Рекомендации по улучшению

Формат ответа: JSON с полями appearance_score, feedback, recommendations

Для анализа рецепта:
text

Сравни описанный рецепт с классическим приготовлением:
1. Соответствие ингредиентов (1-5)
2. Правильность шагов (1-5)  
3. Типичные ошибки
4. Альтернативные варианты

Формат: JSON с recipe_score, missing_ingredients, step_errors, alternatives

Обработка ошибок
Типичные сценарии:

    Плохое качество фото → Предложить переснять

    Слишком короткий рецепт → Требовать подробности

    Ошибка сети при анализе → Сохранить черновик

    Долгая обработка AI → Показать прогресс

Fallback стратегии:

    Локальная обработка если AI недоступен

    Кэширование результатов для похожих блюд

    Деградация до базовой оценки без AI

Оптимизации:

    Ленивая загрузка AI компонентов

    Кэширование результатов анализа

    Пререндеринг статических рекомендаций

    Инкрементальная отдача результатов

Метрики:

    Время анализа: < 30 секунд

    Точность оценок: > 85%

    Удовлетворенность пользователей: > 4/5

## 5. Проблемы и их решения

### Проблема 1: Установка зависимостей Python
**Описание:** Ошибки при установке psycopg2 и pydantic на Python 3.13
**Решение:** 
- Установка системных зависимостей (python3-dev, libpq-dev)
- Использование Python 3.11 для совместимости
- Альтернативные пакеты (psycopg3 вместо psycopg2)

### Проблема 2: Отсутствие bolt.ai/V0
**Описание:** Нет доступа к указанным инструментам прототипирования
**Решение:** 
- Создание собственных AI компонентов
- Демо-страница для показа возможностей
- Подробная документация процесса

### Проблема 3: Интеграция фронтенда и бэкенда
**Описание:** Разные порты, CORS политики
**Решение:**
- Настройка CORS middleware в FastAPI
- Проксирование запросов в development
- Единая структура API
