# Руководство по оптимизации ML моделей для оценки чистоты

## Введение

Это руководство документирует best practices и learnings из A/B тестирования промптов для модели LLaVA, используемой для оценки чистоты квартир.

## Архитектурные принципы

### 1. Модель как сервис (MaaS)
- **Асинхронная обработка**: Использование asyncio для неблокирующих запросов
- **Кэширование результатов**: Redis для хранения частых запросов
- **Rate limiting**: Защита от злоупотреблений
- **Health checks**: Мониторинг доступности модели

### 2. Оптимизация промптов

#### Ключевые инсайты:
1. **Длина имеет значение**: Оптимально 80-120 слов
2. **Структура важнее объема**: Четкая структура > подробное описание
3. **Примеры улучшают качество**: Конкретные примеры повышают accuracy
4. **Формат ответа критичен**: Строгий формат упрощает парсинг
