# Требования к модели

## Цель
Разработать ML-систему моделей для автоматического анализа соответствия кандидатов вакансиям и генерации персонализированных вопросов для интервью.

## Типы моделей и роли

### Модель для анализа резюме и вакансий

**Назначение:** Структурированный анализ резюме и вакансий для оценки соответствия кандидатов

**Вывод:** JSON-структура с извлеченными навыками, опытом, образованием и расчетом скоринга

**Модель:** DeepSeek-Coder-V2-Lite (16B)

**Обоснование выбора:**
- **Поддержка русского языка** - оптимизирована для работы с русскоязычными текстами
- **Качество структурированного вывода** - отличные возможности генерации JSON-формата
- **Эффективность кода** - способность точно следовать шаблонам извлечения данных
- **Баланс размера и качества** - 16B параметров достаточно для точного анализа, но не чрезмерно для ресурсов
- **Квантование** - поддерживает 4-bit и 8-bit квантование для оптимизации памяти

### Модель для генерации вопросов

**Назначение:** Генерация персонализированных вопросов для интервью на основе анализа резюме

**Вывод:** Текстовые вопросы различных типов (технические, поведенческие, кейсовые)

**Модель:** OpenChat-3.5-7B

**Обоснование выбора:**
- **Инструктивное следование** - специализирована на точном выполнении инструкций
- **Качество диалога** - оптимизирована для генерации естественных вопросов
- **Креативность** - способность создавать разнообразные формулировки
- **Адаптивность** - может адаптировать стиль вопросов под уровень кандидата
- **Эффективность** - 7B параметров обеспечивают хорошее качество при разумных вычислительных затратах

##  Метрики качества

### Анализ резюме

- **Accuracy извлечения навыков** - точность определения технических и профессиональных навыков из текста резюме
- **F1-score соответствия опыта** - сбалансированная метрика оценки соответствия опыта работы требованиям вакансии
- **Precision определения образования** - точность идентификации учебных заведений и степеней
- **Recall ключевых компетенций** - полнота выявления всех значимых компетенций кандидата

### Генерация вопросов
- **ROUGE-L для релевантности** - оценка смысловой релевантности сгенерированных вопросов контексту резюме
- **BLEU-4 для качества текста** - оценка грамматической корректности и естественности формулировок вопросов
- **Human evaluation score** - субъективная оценка качества вопросов экспертами-рекрутерами по 5-балльной шкале
- **Diversity вопросов** - мера разнообразия типов и тематик сгенерированных вопросов

## Ограничения

### Память и вычислительные ресурсы

| Ресурс | Ограничение | Компенсирующие меры |
|--------|-------------|---------------------|
| Общая RAM | < 16 GB | Квантование 4-bit, потоковая обработка |
| VRAM (опционально) | < 8 GB | CPU-инференс, модель 3B |
| Размер моделей на диске | < 15 GB | Квантование, shared weights |
| Контекстное окно | 8K-32K токенов | Чанкование, суммаризация |

### Стоимость инфраструктуры

| Компонент | Месячная стоимость |
|-----------|-------------------|
| Виртуальная машина (8 CPU, 16GB RAM) | $50-80 |
| Хранилище моделей (50 GB) | $5-10 |
| Энергопотребление | $10-20 |
| Итого | $65-110/месяц |

### Приватность и безопасность

- **Полная локальность**: Нет внешних API вызовов
- **Шифрование данных**: AES-256 при хранении
- **Изоляция**: Docker-контейнеризация ML-сервисов
- **Аудиторские логи**: Только мета-данные запросов
- **Соответствие**: 152-ФЗ (все данные в РФ)

### Функциональные ограничения

- **Максимальный размер резюме**: 5 страниц
- **Поддерживаемые языки**: Русский (приоритет), Английский (базово)
- **Форматы**: PDF, DOCX, TXT (ограниченная поддержка сложного форматирования)
- **Максимум одновременно обрабатываемых резюме**: 20
- **Ограничение контекста**: 8K токенов на анализ
